# Polindo Frontend

A modern, SEO-optimized blog platform built with Nuxt 4, Vue 3, and TypeScript.

## Tech Stack

- **Framework**: Nuxt 4 (v4.1.3) with Vue 3 (v3.5.22)
- **Language**: TypeScript (strict mode enabled)
- **Styling**: Tailwind CSS with custom configuration
- **Plugins**: @tailwindcss/typography, @tailwindcss/forms
- **Font**: Inter (via @nuxt/fonts)

## Documentation

- [Nuxt 4 Documentation](https://nuxt.com/docs/getting-started/introduction)
- [Nuxt 4 Deployment Guide](https://nuxt.com/docs/4.x/getting-started/deployment#nodejs-server)

## Setup

Make sure to install dependencies:

```bash
# npm
npm install

# pnpm
pnpm install

# yarn
yarn install

# bun
bun install
```

## Development Server

Start the development server on `http://localhost:3000`:

```bash
# npm
npm run dev

# pnpm
pnpm dev

# yarn
yarn dev

# bun
bun run dev
```

## Type Checking

Run TypeScript type checking:

```bash
npm run type-check
```

## Production

### Build for Production

Build the application for production:

```bash
# npm
npm run build

# pnpm
pnpm build

# yarn
yarn build

# bun
bun run build
```

This command will generate the `.output` directory with the production-ready application.

### Run Production Server (Node.js)

After building, start the production server with Node.js:

```bash
# npm
npm run start

# pnpm
pnpm start

# yarn
yarn start

# bun
bun run start
```

The production server will start on `http://localhost:3000`.

**Note**: The `start` script runs `node .output/server/index.mjs`, which is the optimized Node.js server generated by Nuxt. For more information about Node.js deployment, check the [official Nuxt deployment documentation](https://nuxt.com/docs/4.x/getting-started/deployment#nodejs-server).

### Preview Production Build (Development Mode)

Locally preview production build (with HMR for testing):

```bash
# npm
npm run preview

# pnpm
pnpm preview

# yarn
yarn preview

# bun
bun run preview
```

## Docker Deployment

### Build Docker Image

```bash
docker build -t polindo-fe:latest .
```

### Run with Docker Compose

```bash
docker-compose up -d
```

The application will be available at `http://localhost:3001` (mapped from container port 3000).

### Run Docker Container Manually

```bash
docker run -d -p 3001:3000 --name polindo-fe polindo-fe:latest
```

### Docker Commands

```bash
# Stop container
docker-compose down

# View logs
docker-compose logs -f

# Rebuild and restart
docker-compose up -d --build
```

## Environment Variables

Create a `.env` file in the root directory for environment-specific configuration:

```env
# API Configuration (if applicable)
NUXT_PUBLIC_API_BASE_URL=https://api.example.com

# Node.js Server Configuration
NODE_ENV=production
PORT=3000
```
